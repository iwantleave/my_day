# 客户人脉管理系统 — 产品需求文档

## 1. 概述
构建一个面向销售人员的轻量级客户人脉管理工具，帮助记录、跟踪和管理客户关系，提升销售跟进效率。

## 2. 目标用户
- 销售代表 / 客户经理
- 需要维护大量客户人脉的商务人员

## 3. 功能需求

### 3.1 客户管理
- **添加客户**：录入新客户信息
- **编辑客户**：修改已有客户信息
- **删除客户**：删除前需二次确认，防止误删
- **客户列表**：以表格形式展示所有客户，支持快速浏览

### 3.2 客户信息字段
| 字段 | 说明 | 必填 |
|------|------|------|
| 姓名 | 客户姓名 | 是 |
| 性别 | 男 / 女 / 其他 | 否 |
| 生日 | 出生日期 | 否 |
| 年龄 | 按生日计算后存储，编辑时自动更新 | 否 |
| 去世时间 | 客户去世日期 | 否 |
| 公司 | 客户所在公司 | 否 |
| 部门 | 所在部门 | 否 |
| 职位 | 客户职位 | 否 |
| 手机 | 联系电话 | 否 |
| 微信 | 微信号 | 否 |
| QQ | QQ号 | 否 |
| 邮箱 | 电子邮箱 | 否 |
| 地址 | 所在地/办公地址 | 否 |
| 来源 | 怎么认识的（如：展会、朋友介绍、线上广告、陌拜、线上咨询等） | 否 |
| 客户状态 | 潜在客户 / 意向客户 / 谈判中 / 已成交 / 流失 | 否 |
| 客户等级 | A(高意向) / B(中意向) / C(低意向) | 否 |
| 意向产品 | 客户感兴趣的产品/服务 | 否 |
| 预计金额 | 预计成交金额(元) | 否 |
| 预计成交日期 | 预期签单时间 | 否 |
| 负责销售 | 跟进人/销售员 | 否 |
| 下次跟进 | 计划下次联系时间 | 否 |
| 标签 | 自定义标签（如：VIP、重要、需跟进等） | 否 |
| 备注 | 其他备注信息 | 否 |

### 3.3 搜索与筛选
- **搜索**：按姓名、公司、手机、微信等关键词实时搜索
- **筛选**：按客户状态、客户等级、来源等条件筛选列表
- **排序**：按最后联系时间、预计金额、姓名、创建时间等排序

### 3.4 联系记录
- 每次联系客户时记录详细信息
- 每个客户可查看历史联系时间线
- 记录字段：
  - 联系时间（必填）
  - 联系方式：电话 / 微信 / 邮件 / 面谈 / 短信 / 其他
  - 沟通摘要（必填）
  - 联系结果：有意向 / 暂不需要 / 约见成功 / 已拒绝 / 其他
  - 下次跟进时间（可选）

### 3.5 客户详情弹窗
- 点击客户列表中的客户，弹出详情窗口
- 展示客户全部信息（只读）
- 展示该客户的历史联系记录时间线
- 可在此处添加新的联系记录
- 可在此处编辑客户信息

### 3.6 仪表盘 / 数据概览
- 总客户数
- 各状态客户数量分布（潜在 / 意向 / 谈判中 / 已成交 / 流失）
- 今日需跟进客户（下次跟进时间为今天的客户）
- 逾期未跟进客户（下次跟进时间已过且未完成的）

### 3.7 数据管理
- **导出**：导出客户列表为 CSV 格式
- **数据备份**：拷贝 `crm.db` 文件即完成备份
- **数据恢复**：替换 `crm.db` 文件即完成恢复

## 4. 用户界面

### 4.1 仪表盘概览页
- 顶部：标题 + 统计数据卡片（总客户数、今日需跟进、逾期未跟进）
- 中部：客户状态分布图（各状态数量）
- 下方：今日需跟进客户列表快捷入口

### 4.2 客户列表页
- 顶部：标题栏 + 搜索框 + 筛选器 + 添加客户按钮 + 导出按钮
- 主体：客户表格，每行展示姓名、公司、职位、客户等级、客户状态、最后联系时间、下次跟进时间
- 底部：分页或统计总数

### 4.3 添加/编辑客户弹窗
- 表单弹窗，包含所有信息字段，分基本信息、联系方式、销售信息等区域
- 生日使用 date_input，年龄自动计算显示
- 保存后刷新列表

### 4.4 客户详情弹窗
- 上方：客户基本信息卡片（全部字段展示）
- 中部：联系记录时间线，按时间倒序排列
- 下方：添加联系记录按钮 + 编辑客户按钮

### 4.5 删除确认弹窗
- 删除客户前弹出确认框，显示客户姓名，防止误删

## 5. 软件架构

### 5.1 项目结构（扁平化）
```
myday/
├── main.py              # 入口，页面路由，应用启动
├── models.py            # 数据模型 (Customer, ContactRecord 结构定义)
├── database.py          # SQLite 数据库操作（建表、CRUD）
├── service.py           # 业务逻辑层（搜索、筛选、统计、年龄计算等）
├── ui_dashboard.py      # 仪表盘概览页
├── ui_list.py           # 客户列表页（含搜索与筛选）
├── ui_detail.py         # 客户详情弹窗 + 联系记录时间线
├── ui_form.py           # 添加/编辑客户表单弹窗
├── crm.db               # SQLite 数据库文件（自动生成）
└── PRD.md
```

### 5.2 分层职责
| 层 | 文件 | 职责 |
|----|------|------|
| 数据模型 | `models.py` | Customer、ContactRecord 等数据类的定义，不含逻辑 |
| 数据访问 | `database.py` | SQLite 建表、增删改查，返回字典/列表 |
| 业务逻辑 | `service.py` | 调用 database 层，实现搜索、筛选、统计、年龄计算、导出等 |
| 界面层 | `ui_*.py` | NiceGUI 页面和弹窗，调用 service 层获取/操作数据 |
| 入口 | `main.py` | 路由注册，启动应用 |

### 5.3 数据流
```
UI (ui_*.py) → Service (service.py) → Database (database.py) → SQLite (crm.db)
```

## 6. 技术选型
- 框架：NiceGUI (Python)
- 数据存储：SQLite（Python 内置 sqlite3，零安装依赖）
- 数据库文件：`crm.db`，单文件便携，拷贝即备份
- 界面：单页应用，所有操作为弹窗/对话框
- 列表组件：`ui.table`（基于 Quasar QTable，内置排序、分页、行点击）
- 弹窗组件：`ui.dialog` + `ui.card`

## 7. 数据库设计

### 7.1 客户表 (customers)
```sql
CREATE TABLE customers (
    id           TEXT PRIMARY KEY,
    name         TEXT NOT NULL,
    gender       TEXT,
    birthday     TEXT,
    age          INTEGER,
    death_date   TEXT,
    company      TEXT,
    department   TEXT,
    position     TEXT,
    phone        TEXT,
    wechat       TEXT,
    qq           TEXT,
    email        TEXT,
    address      TEXT,
    source       TEXT,
    status       TEXT DEFAULT '潜在客户',
    level        TEXT DEFAULT 'C',
    product      TEXT,
    amount       REAL,
    deal_date    TEXT,
    sales        TEXT,
    next_follow  TEXT,
    tags         TEXT,
    remark       TEXT,
    created_at   TEXT,
    updated_at   TEXT
);
```

### 7.3 说明
- `last_contact`（最后联系时间）：不冗余存储，列表展示时通过 SQL 查询 `contacts` 表按客户分组取 `MAX(time)` 获得
- `created_at` / `updated_at`：由代码自动维护，新增时写 `created_at` 和 `updated_at`，编辑时只更新 `updated_at`
- 年龄：存入数据库，编辑客户时根据生日自动计算并存储；可通过批量操作刷新所有客户年龄
- SQLite 外键约束需执行 `PRAGMA foreign_keys = ON` 生效

### 7.2 联系记录表 (contacts)
```sql
CREATE TABLE contacts (
    id           TEXT PRIMARY KEY,
    customer_id  TEXT NOT NULL REFERENCES customers(id) ON DELETE CASCADE,
    time         TEXT NOT NULL,
    method       TEXT,
    summary      TEXT NOT NULL,
    result       TEXT,
    next_follow  TEXT
);
```
